    Detailed Case Study: Dhaal Pay

    Introduction

    Project Title: Dhaal Pay

    Dhaal Pay is a mobile application designed to empower users in the realm of digital payments by ensuring security and offering comprehensive financial management tools. The app enables users to verify UPI IDs before making payments and provides additional features such as an expense manager, payment due reminders, and CIBIL score insights.

    Role: Product Designer & Developer

    As the primary creator, I oversaw the end-to-end development of Dhaal Pay, including user research, interface design, prototyping, development, and deployment.

    Timeline: 4 months (September 2024 – December 2024)

    Tools Used: Figma, Flutter, Firebase, Google APIs, Adobe Illustrator

    Problem Statement

    The Challenge:

    • Validate the authenticity of UPI IDs before making transactions.
    • Gain clarity about a vendor’s transaction patterns.
    • Manage their finances effectively without switching between multiple apps.

    Key Questions:

    • How can we make UPI payments safer and more reliable for users?
    • How can we integrate financial tools into a payment app without overwhelming users?
    • What are the critical pain points users face while managing personal finances and digital transactions?

    Research

    User Research:

    To gain insights, I conducted interviews and surveys with 30 potential users, focusing on:

    • Their experiences with UPI payments.
    • Issues faced with fraudulent transactions.
    • Pain points in managing finances using existing tools.

    Key Findings:

    • UPI Fraud Concerns:
      • 65% of users worried about the authenticity of UPI IDs when transacting with unfamiliar vendors.
      • Many users lacked a way to verify transaction security.
    • Need for Financial Management Tools:
      • 72% expressed frustration with fragmented apps for expense tracking and reminders.
      • Users wanted an integrated solution for managing due payments and tracking their financial health.
    • User Preferences: A significant preference for intuitive, minimalistic designs with clear visual cues for financial insights.

    Ideation & Planning

    Goals:

    • Build a feature that allows users to verify UPI IDs with critical details like transaction frequency, UPI age, security status, and vendor’s approximate location.
    • Introduce tools for tracking expenses, setting up payment reminders, and checking/improving CIBIL scores.
    • Ensure seamless integration with existing UPI apps for payments.

    Core Features Identified:

    • UPI Verification System
    • Financial Dashboard (Expense Manager and Payment Reminders)
    • CIBIL Score Checking and Improvement Tips

    Design Process

    Wireframing & Prototyping:

    Low-fidelity wireframes were sketched to map the app’s core functionalities. The user flow focused on ease of navigation across these key features:

    • UPI Verification Flow: Users enter the UPI ID, and the system displays details such as transaction volume, UPI age, and security status.
    • Expense Management Dashboard: A central view showing categorized expenses and payment reminders.
    • CIBIL Insights Section: Displays the user’s CIBIL score with personalized recommendations.

    UI Design:

    • Color Scheme: A secure and trustworthy palette with shades of blue and green.
    • Typography: Clear and legible fonts to ensure accessibility.
    • Visual Elements: Icons and charts for quick data comprehension.

    Key Features

    • UPI Verification: Allows users to verify UPI IDs with details like transaction frequency, UPI age, security status, and vendor location.
    • Payment Redirection: Redirect payments to popular UPI apps like Google Pay and PhonePe.
    • Expense Manager: Tracks expenses with visual breakdowns and automatic categorization.
    • Payment Due Reminders: Sends notifications for upcoming bills and dues.
    • CIBIL Score Management: Displays real-time CIBIL scores and offers actionable improvement tips.

    Development Process

    Technology Stack:

    • Frontend: Flutter for cross-platform development.
    • Backend: Firebase for real-time database and authentication.
    • APIs: Google Maps API for location data, third-party APIs for CIBIL score integration.

    Challenges Faced:

    • Real-Time UPI Data Integration: Fetching and analyzing transaction patterns was complex.
    • CIBIL Score Integration: Ensuring accurate data while adhering to financial regulations.
    • User-Friendly Design: Presenting detailed information without overwhelming users required multiple iterations.

    Results

    Quantitative Outcomes:

    • User Adoption: 2,000+ downloads in the first month of launch.
    • Fraud Reduction: 80% of users reported increased confidence in UPI transactions.
    • Engagement: 65% of users actively used the expense manager and payment reminders.

    Qualitative Feedback:

    Users praised the app’s simplicity and the UPI verification feature as a game-changer. Financial insights and reminders were highlighted as standout features for improving personal finance management.

    Future Scope

    • AI-Powered Fraud Detection: Predict and flag potentially fraudulent UPI IDs using machine learning.
    • Multi-Language Support: Expand accessibility for regional audiences.
    • Advanced Financial Tools: Add investment tracking, budgeting, and savings goal tracker features.

    Conclusion

    Dhaal Pay bridges the gap between digital payment security and financial management. By addressing user pain points, the app has set a new standard for safer and smarter payment platforms. This project has been a valuable learning experience, reinforcing the importance of user-centric design and the potential for technology to solve real-world problems.
